# Техническое Задание: Музыкальный Плейлист-Коллаборатор (Yandex Music Edition)

**Версия:** 3.0 (Yandex Music Real API)  
**Срок реализации:** 2–3 недели (Sprint)  
**Команда:** 2 разработчика (Fullstack JavaScript)  
**Язык разработки:** JavaScript (ES6+)  

## 1. Описание проекта
Веб-приложение для совместного управления музыкальным плейлистом в реальном времени с интеграцией **Яндекс Музыки**. Пользователи создают «Комнаты», приглашают друзей, находят треки через API Яндекс Музыки, голосуют за них и слушают музыку синхронно. Треки в очереди сортируются автоматически на основе рейтинга.

**Ключевые отличия от предыдущих версий:**
1.  **Реальное API:** Используется Яндекс Музыка API (метаданные + поиск) вместо моков.
2.  **Воспроизведение:** Реализация возможности прослушивания треков (через виджет/плеер Яндекс).
3.  **Сохранение:** Возможность добавить понравившиеся треки из очереди в личный плейлист пользователя Яндекс.Музыки.
4.  **Стек:** Чистый JavaScript (без TypeScript).

## 2. Роли пользователей
### Создатель (Host)
*   Создает комнату.
*   Управляет статусом воспроизведения (Play/Pause/Next) — синхронизирует плеер для всех.
*   Может удалять любые треки из очереди.
*   Может завершить комнату (kick all).

### Участник (Guest)
*   Присоединяется по коду комнаты.
*   Может добавлять треки в очередь (через поиск Яндекс).
*   Может голосовать (Like/Dislike).
*   Видит очередь и текущий трек.
*   Может добавить трек из очереди в свой личный плейлист Яндекс.Музыки.

## 3. Функциональные требования

### 3.1. Авторизация (Yandex OAuth 2.0)
*   Вход через кнопку «Войти через Яндекс».
*   **Flow:** Authorization Code Flow.
*   **Scopes (Права доступа):**
    *   `login:info` — получение имени и аватара.
    *   `playlist:read` — чтение любимых треков (опционально).
    *   `playlist:write` — добавление треков в плейлист пользователя.
*   **Сессия:** JWT токен или сессия на сервере, привязанная к `yandex_user_id`.
*   **Хранение токенов:** `access_token` и `refresh_token` хранятся на бэкенде (в зашифрованном виде) для доступа к API от имени пользователя.

### 3.2. Комнаты
*   Создание комнаты: генерация уникального 6-значного кода.
*   Присоединение: ввод кода.
*   Отображение списка участников в комнате (аватары из Яндекса).
*   Автоматическое удаление комнаты при выходе Host (или передача прав).

### 3.3. Поиск и Добавление треков (Yandex API)
*   Поиск треков по названию/исполнителю через API Яндекс Музыки.
*   Отображение результатов: Обложка, Название, Артист, Длительность.
*   Добавление в «Очередь» (Queue).
*   Проверка на дубликаты (не добавлять трек с тем же `track_id` в активную очередь).

### 3.4. Голосование и Очередь
*   **Механика:** Score = Likes - Dislikes.
*   **Сортировка:**
    1.  Статус `playing` (всегда первый).
    2.  По убыванию `score`.
    3.  По времени добавления (FIFO) при равном score.
*   **Real-time:** Обновление очереди у всех пользователей при изменении голоса (Socket.io).
*   **Ограничение:** Один голос от одного `yandex_user_id` за один трек.

### 3.5. Плеер и Воспроизведение
*   **Интеграция:** Использование официального виджета Яндекс Музыки или API для управления воспроизведением.
*   **Синхронизация:** Когда Host нажимает Play/Pause/Skip, состояние меняется у всех участников.
*   **Отображение:** Обложка текущего трека, прогресс-бар (синхронизированный).
*   *Техническое ограничение:* Прямые ссылки на потоки (stream URLs) могут быть защищены. Рекомендуется использовать iframe-виджет Яндекс Музыки, управляемый через postMessage, либо preview-версии треков, если доступны через API. В рамках ТЗ предполагается интеграция виджета для легального прослушивания.

### 3.6. Личный плейлист (Save Feature)
*   Кнопка «Добавить в мой плейлист» рядом с треком в очереди.
*   При нажатии бэкенд использует `access_token` пользователя для вызова API Яндекс Музыки (добавление трека в библиотеку/плейлист).
*   Уведомление об успехе/ошибке.

## 4. Технический стек
*   **Frontend:** React (Vite) + JavaScript.
*   **State Management:** Zustand или Context API + React Query.
*   **Backend:** Node.js (Express или NestJS) + JavaScript.
*   **Real-time:** Socket.io.
*   **Database:** PostgreSQL (основные данные) + Redis (опционально для кэша сессий).
*   **API:** Yandex Music API (официальный или стабильный wrapper, например, `yandex-music-api`).
*   **Контейнеризация:** Docker & Docker Compose.
*   **Linting:** ESLint + Prettier (строгие правила для JS).

## 5. Архитектура и Модель Данных

### 5.1. Схема БД (PostgreSQL)
*   `Users`:
    *   `id` (PK), `yandex_id` (unique), `nickname`, `avatar_url`, `access_token` (encrypted), `refresh_token` (encrypted), `token_expires_at`.
*   `Rooms`:
    *   `id` (PK), `code` (unique, index), `host_id` (FK -> Users), `created_at`, `is_active`.
*   `Tracks` (Кэш метаданных):
    *   `id` (PK), `yandex_track_id`, `title`, `artist`, `cover_url`, `duration`.
*   `Room_Tracks` (Queue):
    *   `id` (PK), `room_id` (FK), `track_id` (FK), `added_by_user_id` (FK), `score` (default 0), `added_at`, `status` (queued, playing, played).
*   `Votes`:
    *   `id` (PK), `room_track_id` (FK), `user_id` (FK), `value` (+1/-1), `created_at`.
    *   *Unique constraint:* `room_track_id` + `user_id`.

### 5.2. API Endpoints (REST)
*   `GET /auth/yandex` — редирект на OAuth Яндекс.
*   `GET /auth/callback` — обработка кода, сохранение токенов, редирект в приложение.
*   `GET /auth/me` — данные текущего пользователя.
*   `GET /tracks/search?q=` — прокси-запрос к API Яндекс (чтобы не светить ключи на клиенте).
*   `POST /rooms` — создать комнату.
*   `POST /rooms/:code/join` — войти в комнату.
*   `POST /rooms/:id/tracks` — добавить трек в очередь.
*   `POST /tracks/:id/save` — добавить трек в личный плейлист пользователя (использует токен пользователя).

### 5.3. WebSocket Events (Socket.io)
*   `join_room` / `leave_room`.
*   `queue_update` — сервер рассылает обновленный список очереди (сортированный).
*   `vote_track` — клиент отправляет голос, сервер пересчитывает.
*   `player_state` — изменение статуса плеера (Play/Pause/Skip) от Host.
*   `track_changed` — уведомление о переходе на следующий трек.

## 6. Специфика интеграции Яндекс Музыки
1.  **Регистрация приложения:** В Яндекс OAuth (https://oauth.yandex.ru/client/new).
2.  **Redirect URI:** `http://localhost:3000/auth/callback` (dev), `https://domain.com/auth/callback` (prod).
3.  **API Лимиты:** Учесть rate-limits Яндекс API. Кэшировать поисковые запросы на бэкенде (Redis) на 5-10 минут.
4.  **Воспроизведение:**
    *   Вариант А (Рекомендуемый): Встраивание `<iframe>` плеера Яндекс Музыки. Управление через JS (play/pause) ограничено политиками браузера и CORS.
    *   Вариант Б (Для демо): Использование preview-URL (если доступны через API wrapper) для 30-секундных отрывков.
    *   *В ТЗ фиксируем:* Использование виджета для полноценного прослушивания, синхронизация состояния (какой трек играет) через Socket.io, даже если управление самим аудио-потоком происходит на стороне клиента через виджет.

## 8. Критерии приемки (Definition of Done)
1.  **Авторизация:** Вход через Яндекс работает, профиль подтягивается, токены обновляются.
2.  **Комнаты:** Создание, вход по коду, отображение участников.
3.  **Поиск:** Поиск возвращает реальные треки Яндекс Музыки.
4.  **Очередь:** Добавление трека видно всем без перезагрузки. Голоса меняют сортировку мгновенно.
5.  **Плеер:** Можно запустить воспроизведение текущего трека (через виджет/интеграцию).
6.  **Плейлист:** Кнопка «Добавить в плейлист» реально добавляет трек в библиотеку Яндекс пользователя.
7.  **Код:** Отсутствие `console.log`, код отформатирован (Prettier), нет критических уязвимостей (токены не в коде).
8.  **Документация:** README с инструкцией по настройке OAuth в Яндексе и запуску (`npm install`, `docker-compose up`).

## Приложение Б: Логика добавления в плейлист (Backend)
1.  Получить `room_track_id` из запроса.
2.  Найти трек в БД, получить `yandex_track_id`.
3.  Найти пользователя в БД, получить `access_token`.
4.  Если токен истек -> обновить через `refresh_token`.
5.  Отправить POST запрос к API Яндекс Музыки (`/users/:user_id/playlists/:playlist_id/add-tracks`).
6.  Вернуть статус 200/400 клиенту.
