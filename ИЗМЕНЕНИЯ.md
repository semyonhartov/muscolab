## 🔧 Что нужно настроить перед запуском

### 1. Яндекс OAuth (ОБЯЗАТЕЛЬНО)
1. Перейти на https://oauth.yandex.ru/client/new
2. Создать новое приложение
3. Указать Redirect URI: `http://localhost:3000/auth/callback`
4. Выбрать права: `login:info`, `login:avatar`, `playlist:write`
5. Получить Client ID и Client Secret
6. Вставить в файл `.env`:
   ```
   YANDEX_CLIENT_ID=ваш_id
   YANDEX_CLIENT_SECRET=ваш_secret
   ```

### 2. База данных
**Вариант A: Docker**
```bash
cd back-end
docker compose up -d
```

**Вариант B: Локальный PostgreSQL**
- Создать БД `collab_db` и пользователя `collab_user`
- Выполнить скрипт `001_init.sql`

### 3. Запуск сервера
```bash
cd back-end/server
npm install
npm run dev
```

### 4. Запуск фронтенда
```bash
# Вариант A: Live Server (VS Code)
Открыть front-end/public/index.html через Live Server

# Вариант B: HTTP сервер
cd front-end/public
npx http-server -p 3000
```

## 🎯 Проверка работы

1. **Авторизация**: Вход через Яндекс работает → перенаправление на OAuth → возврат с токеном
2. **Комнаты**: Создание комнаты → генерация 6-значного кода → вход по коду
3. **Поиск**: Ввод запроса → отображение результатов из Яндекс API
4. **Очередь**: Добавление трека → обновление у всех в комнате через Socket.io
5. **Голосование**: Лайк/дизлайк → изменение score → пересортировка очереди
6. **Плеер**: Play/Pause/Next → синхронизация состояния через Socket.io
7. **Сохранение**: Кнопка "Сохранить" → добавление в плейлист Яндекс

## 🐛 Известные ограничения

1. **Воспроизведение музыки**: Используется iframe-виджет Яндекс Музыки или превью. Прямые ссылки на треки могут быть недоступны из-за ограничений API.
2. **Синхронизация**: Синхронизируется состояние (какой трек, пауза/плей), но не аудио-поток. Каждый клиент воспроизводит у себя.
3. **Docker**: Не установлен на машине пользователя (использовать локальный PostgreSQL).
